Проект по автоматизации тестирования Task Management System

Этот проект содержит набор автоматизированных тестов (UI и API) для системы управления задачами. Тесты написаны на языке Python
с использованием фреймворка Playwright.

**Технологический стек**
Язык: Python 3.10+
Фреймворк: Pytest
Инструмент автоматизации: Playwright 
Отчетность: Allure
Архитектура: Page Object Model (POM) + Fluent API Client

Быстрый запуск

**1.Подготовка окружения**
Убедитесь, что у вас установлен Python. Склонируйте репозиторий и установите зависимости:

Создание виртуального окружения:
python -m venv .venv

Активация окружения (Windows):
.venv\Scripts\activate

Установка библиотек:
pip install -r requirements.txt

Установка браузеров Playwright:
playwright install chromium

**2.Настройка конфигурации**

Создайте файл ".env" в корне проекта (на основе примера) и укажите данные для авторизации:
env
BASE_URL=http://localhost:3000
ADMIN_EMAIL=admin@example.com
ADMIN_PASSWORD=admin123
USER_EMAIL=diana@example.com
USER_PASSWORD=password123


**3. Запуск тестов**

Вы можете запускать тесты целиком или по категориям

**Запуск всех тестов:**
pytest

**Запуск только UI тестов в видимом режиме браузера:**
pytest tests/test_create_board.py --headed

**Запуск только API тестов с выводом логов в консоль:**
pytest tests/api/test_board_api.py -s

**Запуск всех тестов с генирацией отчетов Allure:**
run_tests.bat

**Отчетность (Allure)**
Тесты интегрированы с Allure для генерации детальных отчетов со скриншотами.

**Запуск тестов с сохранением результатов:**

pytest --alluredir=allure-results

**Генерация и открытие отчета:**
 
allure serve allure-results
   
В отчете доступны шаги выполнения (Steps), скриншоты финальных состояний и информация о сетевых запросах.

**Структура проекта:**
pages/ — Page Object классы и API-клиент.
tests/ — UI-сценарии (создание, навигация).
tests/api/ — Тесты серверной логики.
data/ — Хранилище состояний авторизации (session cookies).
conftest.py — Глобальные фикстуры и настройки авторизации.

 
**Архитектура проекта (Page Object Model)**
Проект реализован с использованием паттернов проектирования, обеспечивающих легкую поддержку и масштабируемость кода.

1. BasePage (pages/base_page.py)
Является родительским классом для всех страниц. Содержит общие методы обертки над Playwright:
click(), send_keys(), open() — с автоматическим логированием каждого действия в Allure.
make_screenshot() — метод для создания вложений в отчет.
verify_url() — базовые проверки навигации.

2. BoardPage (pages/create_board_page.py)
Описывает бизнес-логику работы с досками.
Хранит селекторы элементов (кнопки создания, инпуты, чекбоксы) через стабильные атрибуты data-qa.
Содержит высокоуровневые методы, такие как create_board(), скрывая детали реализации (клики, ожидания) от текста самого теста.

3. ApiClient (pages/api_client.py)
Специализированный слой для работы с серверной частью (Backend).
Реализует Fluent Interface: методы post, get, delete принимают параметр expected_status.
Автоматически проверяет коды ответов сервера (Assertion), избавляя тесты от дублирования проверок.
Интегрирован с контекстом браузера для использования актуальных CSRF-токенов и сессионных Cookies.
4. Тестовый слой (tests/)
Тесты максимально абстрагированы от деталей реализации. Они оперируют только бизнес-целями (например, "создать доску под админом").
UI-тесты: Проверяют пользовательские сценарии и корректность отображения данных на экране.
API-тесты: Обеспечивают быструю проверку логики сервера и целостности данных в базе.

**Инструкция по запуску тестов в Docker-контейнере**
Для обеспечения кроссплатформенности и изоляции среды тестирования был создан Docker-образ на базе официального образа Microsoft Playwright.
1. Подготовка окружения:
Перед запуском тестов в контейнере убедитесь, что:
Тестируемое приложение (сайт) запущено на хостовой машине (Windows) и доступно по адресу http://localhost:3000.
В файле pytest.ini отсутствует флаг --headed (для корректной работы в фоновом режиме внутри Linux).
2. Сборка Docker-образа:
Выполните команду в корне проекта для создания образа с именем diploma-tests:
docker build -t diploma-tests .
Используйте код с осторожностью.
3. Запуск тестов
Для запуска используется режим сетевого взаимодействия host, чтобы контейнер имел доступ к локальному серверу Windows:
**Запуск апи тестов в докере:**
 docker run --rm --network host -e PYTHONPATH=/app diploma-tests pytest tests/api/ -c /dev/null -v
**Запуск UI тестов в докере:**
 docker run --rm --network host -e PW_HEADLESS=1 -e PYTHONPATH=/app diploma-tests pytest tests/ui/ -c /dev/null -v -p no:xdist
